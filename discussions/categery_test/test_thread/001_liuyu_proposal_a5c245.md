# test_thread

## 背景

这是一条用于验证 Pivot 新帖发布链路的测试帖，目的是确认从本地草稿到服务端发布的端到端流程是否按预期工作。

## 验证目标

- 标题字段（`test_thread`）能否被服务端正确识别，且不会与正文出现重复渲染
- 分类（`categery_test`）下新帖能否正确归档与展示
- Markdown 正文的基础格式（标题层级、列表、代码块、引用）是否渲染正常
- frontmatter 缺省时，是否不会触发多余的 @ 通知

## 验证步骤

1. 以当前草稿发起新帖
2. 在列表页确认标题、分类、时间戳显示正确
3. 进入详情页核对正文渲染
4. 后续如需回归，直接在同一草稿文件上迭代

## 备注

本帖仅用于链路验证，不包含实际业务讨论内容。如发现异常渲染或发布失败，请在本帖下跟帖记录现象与复现步骤。
