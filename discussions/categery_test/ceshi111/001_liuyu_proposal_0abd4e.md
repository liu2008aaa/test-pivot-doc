# ceshi111

## 背景

本帖用于在 `categery_test` 分类下再发起一条独立的新帖，重点验证**同分类多帖并存**场景下的列表排序、详情页跳转与草稿迭代链路。与此前的 `test_thread` 不同，本次关注点不在首次发布的可达性，而在多帖共存后的稳定性。

## 验证目标

- 同一分类下出现多条主贴时，列表页按时间倒序或既定规则正确排序
- 不同主贴的详情页跳转互不串，评论与正文归属正确
- 草稿文件覆盖式编辑后，服务端能识别为"同一帖子更新"而非新建重复帖
- 无 `mentions` frontmatter 时，发布链路不触发任何飞书推送

## 验证步骤

1. 以当前草稿发布本帖，记录帖子 ID 与创建时间
2. 在分类列表页核对本帖与 `test_thread` 的相对顺序
3. 分别进入两个帖子的详情页，确认标题、正文、作者、时间戳互不交叉
4. 在本地对同一草稿文件进行一次小幅修改，再次触发发布流程，观察是 patch 还是新建
5. 检查通知端在本次发布中保持静默

## 关注点

- **标题唯一性**：服务端使用标题字段兜底，正文内不再重复 `# ceshi111`，发布后在详情页确认不出现双标题
- **分类归档**：`categery_test` 目录下应能看到本帖对应的新 `.md` 源文件，命名格式与既有帖对齐
- **Markdown 基础语法**：本帖覆盖二级标题、无序列表、有序列表、行内代码与加粗，可作为渲染回归的参考样本

## 后续

如发布链路表现符合预期，本帖可作为"多帖并存"场景的基线样本沉淀下来；若出现排序异常、详情页错位或草稿迭代被识别为新建，请直接在本帖下跟帖记录现象、时间戳与复现步骤，便于后续定位。
